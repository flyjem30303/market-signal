# F Requests To A

這份文件記錄產品設計 / UIUX 設計 F 主動向 PM/開發者 A 提出的需求。

## Request 001：建立設計系統基礎

提出者：F  
狀態：Done  
優先度：高

### 背景

目前頁面可用，但視覺層級、色彩、間距與元件一致性仍偏原型。金融工具需要更高的信任感與掃描效率。

### F 的需求

請 A 先建立全站設計系統基礎：

- 色彩 token：背景、文字、靜音文字、線條、主色、成功、警示、風險。
- 字級與行高規則。
- 卡片、按鈕、導覽、排行列、提示區塊的一致樣式。
- Header 更像正式產品。
- 晨報第一屏更有儀表感。

### 驗收標準

- `/briefing` 第一屏看起來像正式金融工具。
- 導覽不再像臨時連結列。
- 色彩不只是一組淡灰加藍色。
- 卡片層級清楚，主要狀態比次要資訊突出。
- 手機版不溢出。

### A 的完成紀錄

- 已建立全站設計 token：背景、surface、文字、靜音文字、線條、主色、成功、警示、風險、陰影。
- Header 已升級為正式產品導覽，包含品牌標記與副標。
- `/briefing` 第一屏已重做為晨報儀表，包含日期、資料品質、模型版本、主市場分數與燈號。
- 卡片、排行列、狀態 badge 與提示區塊已統一視覺層級。
- 已用瀏覽器驗證 `/briefing` 與 `/weekly` 主要布局。

## Request 002：重整資訊密度

提出者：F  
狀態：Open  
優先度：中

### 背景

目前資訊都在，但部分頁面文字與卡片較平均，使用者不容易快速知道哪個最重要。

### F 的需求

請 A 逐步重整：

- 晨報：先狀態、再風險、再觀察清單。
- 股票頁：先分數與風險，再模組，再新聞與回測。
- 週報：先摘要，再排行，再產業觀察。
- 方法論與法務頁：文字頁要像文件，但仍要有清楚閱讀節奏。
