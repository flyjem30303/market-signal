# 漸進式開發路線圖

## Phase 0：規格化與可轉移

目標：讓專案換電腦也能接續。

任務：

- 建立 README。
- 建立 handoff 文件。
- 建立資料庫 schema 草案。
- 建立 sitemap 草案。
- 建立資料來源清單。
- 建立 MVP 任務清單。

完成標準：

- 新電腦拿到資料夾後，能理解目前狀態與下一步。

## Phase 1：可上線 MVP

目標：做出公開網站雛形。

功能：

- 首頁台股總覽。
- 股票搜尋。
- 股票頁。
- 健康度 / 風險度。
- 近 20 / 60 / 120 日變化。
- 新聞信心評論。
- 基礎回測摘要。
- 愛心收藏先存在瀏覽器。

建議技術：

- Next.js
- Supabase PostgreSQL
- Vercel

完成標準：

- 至少 100 檔股票頁可被搜尋引擎收錄。
- 每日資料可以自動更新。

## Phase 2：SEO 與週報

目標：累積自然流量。

頁面：

- `/stocks/2330`
- `/stocks/0050`
- `/industries/semiconductor`
- `/ranking/healthy-uptrend`
- `/ranking/high-risk`
- `/weekly/taiwan-market-signal-yyyy-mm-dd`

完成標準：

- 每週至少 1 篇週報。
- 股票頁不是薄內容，每頁至少有燈號解讀、趨勢圖、新聞評論、回測摘要。

## Phase 3：廣告

目標：開始小規模變現。

任務：

- 申請 Google AdSense。
- 加入隱私權政策。
- 加入使用條款。
- 加入投資免責聲明。
- 控制廣告位置，不破壞信任感。

完成標準：

- 頁面穩定收錄。
- 有穩定自然流量。

## Phase 4：會員

免費會員：

- 雲端愛心清單。
- 最近查看。
- 每週 email 摘要。

付費會員：

- 自訂燈號通知。
- 進階回測。
- 無廣告。
- 每週 PDF 報告。
- 產業排行。

## Phase 5：聯盟行銷

方向：

- 券商開戶。
- 財經書籍。
- 投資工具。
- 課程合作。

原則：

- 不放在個股燈號旁邊。
- 必須明確揭露合作或聯盟關係。
- 優先放在新手教學、工具頁、週報底部。
