# Model Scoring Standard

這份文件由 C：專業投資顧問提出，作為後續正式模型的可信度標準。

## 評分定位

本模型的目標是建立「市場狀態儀表」，不是買賣訊號。

- 多頭健康度：衡量趨勢、基本面、資金與產業支撐是否仍存在。
- 回檔風險度：衡量估值、籌碼、波動、集中度與宏觀壓力是否升高。
- 綜合燈號：把健康度與風險度合併，作為投資節奏參考。

## v1 六大模組

| 模組 | 建議權重 | 主要資料 | 專業判讀 |
| --- | ---: | --- | --- |
| 價格趨勢 | 18% | 收盤價、均線、相對強弱、成交量 | 趨勢是否延續，是否出現跌破結構 |
| 獲利基本面 | 18% | 月營收、EPS、毛利率、營益率 | 獲利是否支撐評價 |
| 估值壓力 | 16% | PE、PB、殖利率、歷史分位 | 是否進入追價或過熱區 |
| 市場廣度 / 族群 | 14% | 同產業上漲比例、站上均線比例 | 漲勢是否擴散，不只靠少數權值股 |
| 籌碼資金 | 16% | 三大法人、融資、借券、當沖比 | 資金是否穩定，散戶槓桿是否偏熱 |
| 宏觀與上游 | 18% | SOX、NASDAQ、VIX、10Y、DXY、USD/TWD | 外部環境是否支持風險資產 |

## 股票與 ETF 應分開校準

個股需要更重視基本面、估值、籌碼與產業資料。

ETF 需要更重視成分股廣度、整體市場趨勢、折溢價、成交量與大盤風險。

## 回測最低標準

每個燈號至少要揭露：

- 樣本期間。
- 樣本數。
- 20 日 / 60 日平均報酬。
- 勝率。
- 最大回檔。
- 是否含交易成本。
- 是否有存活者偏誤。

## 資料品質最低標準

正式模型應顯示資料完整度：

- A：核心資料完整，更新時間正常。
- B：少數非核心資料缺漏，分數可參考。
- C：重要資料缺漏，分數需保守解讀。
- D：資料不足，不應產生正式燈號。

## 目前限制

目前前端仍使用 mock model 與合成報酬，適合驗證產品體驗，不適合當作真實投資模型。正式上線前必須改接真實資料、記錄模型版本，並完成回測揭露。
