# E CEO Strategy

CEO 對本專案的主軸定義：這不是單次查詢股票頁，而是投資人每天或每週會打開的市場狀態儀表。

第一階段從台股與台灣使用者切入，但公司方向不是台灣限定產品。長期必須拓展為全球股市與多國使用者導向的投資節奏工具。

## 北極星

使用者每週主動回來查看自己的關注標的、市場風險與投資節奏；長期可跨市場、跨國家、跨語系使用。

## CEO 判斷

如果網站只靠 SEO 股票頁，流量可能有，但黏著度不足。真正有長期價值的工具應該讓使用者形成習慣：

- 每天看一次：今天市場健康嗎？
- 每週看一次：本週哪些風險升溫？
- 加入愛心：我的關注標的有沒有變化？
- 看方法論：我為什麼能相信這個分數？
- 看週報：這週怎麼整理投資節奏？

## 第一階段產品方向

第一階段仍以台灣市場為 wedge market。理由是台灣資料來源、使用者情境與產品敘事最清楚，能最快驗證留存與資料可信度。

但從現在開始，A 在資料模型、路由、文案、時間、幣別與評分模型上，不得把台灣假設寫死成不可改架構。

1. 每日晨報 `/briefing`
   - 一頁回答今天該注意什麼。
   - 讓使用者不用每次從股票搜尋開始。

2. 個股頁 `/stocks/[symbol]`
   - 查單一標的。
   - 基礎行情資料 + 深度風險解讀入口。
   - 不能只呈現燈號，必須提供價格、漲跌、成交量、估值、股利與新聞等常見股市資料。

3. 週報 `/weekly`
   - 每週回訪與內容沉澱。

4. 方法論 `/methodology`
   - 建立可信度。

5. 收藏與會員
   - 從 localStorage 愛心，逐步升級到會員雲端收藏與 Email。

## 暫緩事項

- 不急著放正式廣告。
- 不急著做付費會員。
- 不急著買新聞 API。
- 不急著承諾真實投資模型，先把資料品質與方法論補齊。
- 不急著做全球所有股票頁；先把台灣資料可信度做深，再逐步加入美股與全球市場。

## CEO 對 A 的指令

先建立每日晨報入口，讓首頁以外有一個「每天會打開」的頁面。此頁先用 mock repository 產生，未來接真實資料後可變成每日自動生成。

第二個指令：股票頁必須從「燈號模型頁」升級為「股市資料頁 + 智能解讀頁」。常見資料是信任基礎，健康度與回檔風險是差異化。

## 董事長可能質疑

- 使用者真的會每天打開嗎？
- 晨報和週報差在哪？
- 沒有真實資料前會不會失去信任？
- 商業化會不會太早？

CEO 必須用產品數據與使用場景回答，而不是只用功能數量回答。

## 全球化決策

董事長要求未來網站必須拓展為全球型產品，CEO 決策如下：

1. 現階段不改成泛全球 MVP，仍先完成台股資料可信度。
2. 從下一個資料庫切片開始，所有 schema 與 repository 設計都要預留全球市場欄位。
3. 股票代號不能被視為全球唯一鍵，未來要以 market / exchange / symbol 或 asset id 處理。
4. 評分模型不能假設所有市場共用同一權重，必須支援 market-specific model version。
5. UI 暫時維持繁中台股體驗，但不得把幣別、時區、交易所與語系寫死在 reusable logic。
