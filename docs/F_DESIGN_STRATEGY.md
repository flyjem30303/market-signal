# F Design Strategy

F：產品設計 / UIUX 設計的核心判斷：目前網站功能方向正確，但視覺專業度不足，會影響金融工具的信任感與長期使用意願。

## 設計目標

讓使用者打開網站時感覺這是一個安靜、可信、可反覆使用的投資儀表，而不是臨時拼出的資料頁。

## 設計原則

1. 資訊先於裝飾
   - 不做花俏 landing page。
   - 第一屏必須讓使用者知道今天市場狀態。

2. 專業但不冰冷
   - 使用乾淨的金融工具風格。
   - 色彩要克制，但燈號與風險要清楚。

3. 高資訊密度但可掃描
   - 儀表板要能快速比較。
   - 長文要切成短段、摘要、排行、表格。

4. 可信度要可見
   - 資料品質、模型版本、免責聲明要容易找到。
   - 但不能壓過主要決策資訊。

5. 手機可讀性優先
   - 手機上卡片順序要符合「先總覽、再風險、再細節」。
   - 表格與排行要避免橫向溢出。

## 第一階段設計方向

- 建立更成熟的色彩系統。
- 重做 header 與頁面第一屏。
- 強化晨報 `/briefing` 作為每日入口。
- 儀表板卡片要有明確層級：主狀態、分數、風險、次要說明。
- 股票頁 SEO 內容要更像研究頁，而不是一般文章。
- 商業合作區要弱化，避免干擾信任感。

## 暫不做

- 不做大型品牌視覺動畫。
- 不做過度行銷式 hero。
- 不做一頁式 landing page。
- 不用過多漸層、陰影與裝飾圖形。

## F 對 A 的第一個要求

先做「設計系統基礎與首頁 / 晨報視覺升級」，不用一次重做全部頁面。目標是讓使用者第一眼感覺更可信、更像成熟金融工具。
